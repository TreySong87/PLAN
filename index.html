1. For What (WHY)

2. HOW?

효율적인 아카이브 구축 및 활용을 위한 구체적인 방법론입니다. 포커 콘텐츠의 특성을 고려하여, 사용자가 방대한 정보 속에서 원하는 자료를 쉽고 빠르게 찾을 수 있도록 지원하는 두 가지 핵심 원칙을 제안합니다.

🎯 원하는 콘텐츠의 명확한 분류

일반 스포츠와 달리 포커는 핸드 정보, 플레이어 액션 등 과다한 정보를 포함하므로, 기획자/작업자가 직관적으로 원하는 것을 찾을 수 있어야 합니다. 이를 위해 메타데이터 태깅을 매우 정밀하게 분류 진행해야 합니다.

예시: NBA 메타데이터 태그

경기 정보

  • 시즌 (2024-2025)
  • 경기 날짜
  • 팀명 (ex. LAL vs BOS)
  • 리그 구분 (정규시즌 / 플레이오프 / 프리시즌 등)
  • 쿼터, 다운, 드라이브, 클러치 타임 등

선수 정보

  • 선수명 (풀네임 + 약어)
  • 선수 번호
  • 포지션 (QB, WR, PF 등)
  • 팀 소속

주요 액션 태그

  • 덩크, 쓰리포인트, 태클, 터치다운, 인터셉션, 블로킹 등
  • 하이라이트 / 슬로우 / 인터뷰 / 벤치 리액션

상황 설명

  • 게임 클로징 / 역전 / 대기록 / 데뷔 / 은퇴 경기
  • 파울 / 비디오 판독 / 논란 상황

감정 기반 태그

  • 감동 / 분노 / 흥분 / 실망 / 리벤지
  • 팬 리액션 / 코치 리액션

포커 메타데이터 태그

영상정보

  • 영상 설명
  • 대회 및 프로젝트 제목
  • 년도
  • 장소(개최장소/대회장/호텔)
  • 사물,풍경,배경 등(B-Roll)
  • 소스(PGM,Clean/Youtube Straming/ESPN Episode 등)

포커플레이정보

  • 플레이어
  • 핸드등급
  • 에픽핸드정보(쿼즈,로얄플러쉬 등)
  • 포커플레이태그(블러프,배드빗,쿨러,등)
  • 토너먼트상황(우승,브레이슬릿,버블 등)
  • 감정기반태그
  • 상황설명

주요핸드스토리태그(핸드의 자세한 분류)

  • 배드빗(다양한 배드빗 분류)
  • 블러프(블러프 분류)
  • 석아웃
  • 쿨러
  • 올인
  • 포스트플랍런아웃

🙌 기획자/작업자의 편의성 증대

아카이브 시스템은 데이터를 저장하는 아카이버가 아닌, 콘텐츠를 직접 사용하는 기획자와 영상 편집자의 작업 효율을 극대화하는 방향으로 설계되어야 합니다. 사용자의 워크플로우를 분석하여 검색, 미리보기, 다운로드 등 모든 과정이 편리하도록 솔루션을 구축하는 것이 목표입니다. 이 부분은 구체적인 협의를 통해 맞춤형으로 설계할 예정입니다.

3. WHEN?

프로젝트의 실행 시기와 일정은 여러 변수에 따라 결정됩니다. 아래 요소를 종합적으로 고려하여 가장 효율적인 계획을 수립하기 위해 추후 심도 있는 논의가 필요합니다.

추후 협의 필요

프로젝트 실행 시기는 도입할 솔루션의 종류, 관리할 데이터의 총 용량, 그리고 프로젝트에 투입될 구성 인원에 따라 크게 달라질 수 있습니다. 각 요소별 세부 사항을 확정한 후 구체적인 일정을 수립할 예정입니다.
